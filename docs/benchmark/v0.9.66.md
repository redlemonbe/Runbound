# Runbound v0.9.66 — XDP fast path: domain-routing ZC fix + wire-builder local-zone

**Date:** 2026-06-03
**Runbound version:** 0.9.66
**Mode:** Bare-metal AF_XDP zero-copy (Drv), Intel X520 (82599) 10 GbE
**Tool:** [dnsmark v1.2.1](https://github.com/redlemonbe/dnsmark)
**Builds on:** [v0.9.65](v0.9.65.md) (first bare-metal XDP 10 G benchmark, 4.77 M qps)

---

## Executive Summary

Two XDP fast-path changes shipped in v0.9.66:

1. **#155 — `xdp-domain-routing` (CPUMAP) no longer breaks zero-copy.** Enabling it on a ZC interface used to drop throughput ~40× (4.77 M → 120 k qps) and route onto HT siblings. `DOMAIN_ROUTING_ENABLED` moved from a frozen `volatile const` to a runtime map flag (`domain_routing_cfg`); it is now forced OFF once zero-copy is confirmed on a socket, so the XDP program falls through to the XSK ZC path. domain-routing remains available in SKB/copy mode (its cache-locality intent).

2. **#156 — hand-rolled wire builder bypasses hickory for local-zone A/AAAA answers.** `answer_dns_wire()` parses the query and builds the response straight into the UMEM frame (no `hickory_proto::Message` parse/serialize, no heap alloc). Everything else (NXDOMAIN, NODATA, wildcards, CNAME/MX/TXT, EDNS, ACL Deny, ANY, malformed) falls back to hickory — identical to previous behaviour, **no regression**.

---

## A/B benchmark — local-zone A answer path

Same query (`a.bench.test A`, a local static A record), X520 10 GbE, dnsmark flood, RSS spread across the 16 NIC-local physical cores.

| | hickory (v0.9.65 / main) | wire builder (#156) | delta |
|---|---|---|---|
| Throughput | 4.11 M qps | **6.48 M qps** | **+58 %** |
| p50 latency | 0.512 ms | **0.335 ms** | **−35 %** |
| p99 latency | 1.745 ms | 2.121 ms | (overload tail) |

The gain is the elimination of hickory's `Message` parse/serialize + heap allocations on the answer hot path. **NXDOMAIN / NODATA / wildcard / non-A-AAAA queries are unchanged** (hickory fallback), so this is a pure win on the A/AAAA answer case with zero behaviour change elsewhere.

## Methodology

- Generator: dnsmark ≥ v1.2.1 (saturates 10 G, per-packet source-port variation).
- Receiver NIC: `ethtool -N <nic> rx-flow-hash udp4 sdfn` (spread UDP across RSS queues) + `ethtool -A <nic> rx off tx off` (no PAUSE throttle).
- `rate-limit: 0`, local-zone static with a specific A record, `xdp: yes`.
- Correctness verified with `dig` (A NOERROR/AA, NXDOMAIN, wildcard via hickory, ACL).

## Notes & follow-up

- The wire builder still calls `hickory Name::read` once per packet to build the `LocalZoneSet` lookup key — now the dominant residual hickory cost. Keying `LocalZoneSet` by the wire QNAME (CRC32) would remove it; profiling on a clean host (perf was unavailable on the bench receiver) is the next step toward higher per-core throughput.
- ZC remains RSS-bound to 16 cores on the 82599 (`ethtool -X equal 40` → EINVAL). Per-core efficiency, not core count, is the lever on this NIC.
- domain-routing (CPUMAP) is intentionally gated OFF on zero-copy interfaces; it is a cache-locality feature for SKB/copy mode, not a throughput-scaling lever.
