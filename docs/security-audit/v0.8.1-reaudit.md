## Methodology

### What was tested

All bypass attempts focused on the specific normalization block added at `src/api/mod.rs:~1604`. The block:

```rust
#[allow(deprecated)]
let ip = if let IpAddr::V6(v6) = ip {
    if v6.is_loopback() { IpAddr::V6(v6) }
    else { v6.to_ipv4().map(IpAddr::V4).unwrap_or(IpAddr::V6(v6)) }
} else {
    ip
};
```

Each bypass attempt was verified by:
1. Static analysis of the Rust normalization path
2. Rust test program compiled on dev VM (192.168.8.245) confirming normalization behavior
3. Production API test on master (192.168.8.12) for key vectors

Additional in-scope: HMAC relay, rate limiter, ring size validation, body size limit.

### What was NOT tested

- DNSSEC validation bypass (requires live DNS with crafted responses)
- TLS TOFU cert fingerprint enforcement (requires network interception)
- Cache timing attacks via DNS response timing differences
- eBPF BPF verifier bypass (requires kernel-level access)
- Supply chain: `cargo audit` run shows only unmaintained warnings, no CVEs
- Other IP-accepting endpoints outside `src/api/mod.rs`

## Bypass Attempts

### Attempt 1 — `::ffff:127.0.0.1` (standard IPv6-mapped loopback)

**Result:** BLOCKED  
`to_ipv4()` normalizes to `127.0.0.1`. `is_loopback()=true`. Request rejected with 400 INVALID_ADDR.  
**Verification:** Production test confirms 400 response.

### Attempt 2 — `::ffff:7f00:0001` (hex form of `::ffff:127.0.0.1`)

**Result:** BLOCKED  
Same normalization path as Attempt 1. Identical result.  
**Note:** This is a sub-case of Attempt 1, not a distinct attack class (see vectors-tested.md).

### Attempt 3 — `::ffff:0.0.0.0` (IPv6-mapped unspecified)

**Result:** BLOCKED  
Normalized to `0.0.0.0`. `is_unspecified()=true`. Request rejected.  
**Verification:** Production test confirms 400 response.

### Attempt 4 — `::127.0.0.1` (deprecated IPv4-compatible, RFC 4291 §2.5.5.1)

**Result:** BLOCKED (after residual fix, commit e938382)  
`to_ipv4()` (deprecated but handles both `::ffff:` and `::` forms) normalizes to `127.0.0.1`. `is_loopback()=true`.  
**Note:** This vector was NOT blocked by the first fix (commit 68712e3) which used `to_ipv4_mapped()` only. It was the primary finding of this re-audit, documented as SEC-2026-05-24-02.  
**Exploitability pre-fix:** Linux routes `::127.0.0.1` to loopback (confirmed via Python socket test on dev VM). Real bypass.

### Attempt 5 — `::ffff:169.254.1.1` (IPv6-mapped link-local)

**Result:** BLOCKED  
Normalized to `169.254.1.1`. Existing `is_link_local()=true` check catches it.

### Attempt 6 — `2002:7f00:0001::` (6to4 embedding of `127.0.0.1`)

**Result:** Not blocked, but not exploitable  
Address remains as IPv6. 6to4 does not route to loopback — it routes via 6to4 relay to an external address. Low practical exploitability in DNS upstream context.  
**Classification:** INFO

### Attempt 7 — X-Forwarded-For rate limit bypass

**Result:** SAFE  
Code explicitly ignores `X-Forwarded-For` (comment VUL-04). API is localhost-only.

### Attempt 8 — HMAC timing oracle in relay

**Result:** SAFE  
`hmac_verify_with_ts()` uses `subtle::ConstantTimeEq`, folded into a single CT accumulator.

### Attempt 9 — Anti-replay window bypass (±30s)

**Result:** SAFE  
Timestamp diff check (`diff > 30`) runs before HMAC comparison. Cannot be bypassed by timestamp manipulation.

### Attempt 10 — Ring size integer overflow

**Result:** SAFE  
`parse_xdp_ring_size()` enforces `is_power_of_two() && (64..=65536)`. Validated with unit tests.

## Fix Verification Status

Both fixes confirmed effective for all tested bypass variants.

**Residual concern (not blocking for v0.8.1):** The fix uses `#[allow(deprecated)]` on `to_ipv4()`. This function is deprecated in Rust's standard library. It functions correctly in Rust 1.95 but may be removed in a future edition. A non-deprecated manual normalization is preferred for long-term stability. Tracked for v0.8.2.

## Verified Safe

- Auth bearer constant-time comparison (`subtle::ConstantTimeEq`) — correct
- HMAC relay verification — constant-time, anti-replay enforced
- Body size limit (65KB) — enforced at axum framework level before deserialization
- Ring size validation — bounds-checked, power-of-2 enforced
- SSRF for standard loopback/unspecified/link-local — blocked
- IPv6-mapped link-local — blocked after normalization
- X-Forwarded-For — correctly ignored

## Known Limitations

1. **Same model family as implementer** — Rule 10 independence requires a different model or human reviewer. Gemini (VM2) was unavailable (quota exceeded). This re-audit was conducted by a separate Claude session, which provides adversarial framing but not model-family independence.
2. **No live DNS testing** — DNSSEC bypass and cache timing attacks require a live DNS environment with crafted responses. Not tested.
3. **No network interception** — TLS TOFU cert fingerprint enforcement cannot be tested without MITM capability.
4. **No kernel access** — eBPF BPF verifier bypass requires kernel-level access. Not tested.
5. **Static analysis only for uncovered endpoints** — Other IP-accepting endpoints (`src/sync/register.rs`, zone SOA, slave sync) were analyzed statically but not tested end-to-end. The same IPv6-mapped normalization gap may exist there.
6. **No supply chain deep audit** — `cargo audit` shows only unmaintained crates warnings (no CVEs). Full dependency tree audit not performed.
