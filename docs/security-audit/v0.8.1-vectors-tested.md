## Honest count notice (AUDIT-PRINCIPLES.md Rule 3)

The release notes for v0.8.1 stated "10 vectors tested." The actual count of **distinct attack classes** is **9**. Attempts 1 and 2 in the raw re-audit (`::ffff:127.0.0.1` and `::ffff:7f00:0001`) are the same technique expressed differently — IPv6-mapped loopback, decimal vs hex. They collapse into one class. The honest count is 9.

## Vectors tested

| # | Attack class | Representative input | Finding | Status |
|---|-------------|----------------------|---------|--------|
| 1 | SSRF via IPv6-mapped loopback | `::ffff:127.0.0.1`, `::ffff:7f00:0001` | SEC-2026-05-24-01 | Fixed in 68712e3 |
| 2 | SSRF via IPv6-mapped unspecified | `::ffff:0.0.0.0` | SEC-2026-05-24-01 | Fixed in 68712e3 |
| 3 | SSRF via deprecated IPv4-compatible loopback | `::127.0.0.1` | SEC-2026-05-24-02 | Fixed in e938382 |
| 4 | SSRF via IPv6-mapped link-local | `::ffff:169.254.1.1` | No finding | Blocked by existing link-local check |
| 5 | SSRF via 6to4 embedding | `2002:7f00:0001::` | INFO | Not a practical vector — routes externally via 6to4 relay |
| 6 | Rate limiter bypass via X-Forwarded-For | `X-Forwarded-For: 1.2.3.4` | No finding | Header explicitly ignored (comment VUL-04) |
| 7 | HMAC timing oracle in relay | Variable-length HMAC comparison | No finding | Uses `subtle::ConstantTimeEq`, single CT accumulator |
| 8 | Anti-replay window bypass | Timestamp ±30s manipulation | No finding | Timestamp diff checked before HMAC comparison |
| 9 | Ring buffer size integer overflow | `ring_size` parameter parsing | No finding | `is_power_of_two() && (64..=65536)` enforced |

## What was NOT tested

- DNSSEC validation bypass (requires live DNS with crafted responses)
- TLS TOFU cert fingerprint enforcement (requires network interception)
- Cache timing via DNS response timing differences
- eBPF BPF verifier bypass (requires kernel-level access)
- Supply chain: `cargo audit` — 2 unmaintained warnings only (paste 1.0.15, rustls-pemfile 2.2.0), no CVEs
- Other IP-accepting endpoints (`src/sync/register.rs`, zone SOA, slave sync) — same IPv6-mapped pattern may be present, tracked for v0.8.2 audit
