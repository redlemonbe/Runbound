## Fix Details

### SEC-19 — CSRF [FIXED]

**Implementation:** `src/webui/mod.rs`

Login handler generates a 32-hex CSRF token via `uuid::Uuid::new_v4()`, stores it as
`(expiry, csrf_token)` in the sessions DashMap, and sets it as a non-HttpOnly cookie
`rb_csrf`. `change_password()` and `handle_logout()` (POST) call `verify_csrf()` which
reads `x-csrf-token` from request headers and compares to the session-stored token.

The JS `api()` helper reads `rb_csrf` via `document.cookie` and injects `X-CSRF-Token`
on all non-GET/HEAD requests (`examples/web-ui/index.html`).

**Residual risk:** `verify_csrf()` returns `false` when no session is active (no bypass
possible). The `SameSite=Lax` attribute remains; the CSRF token is the primary control.

---

### SEC-AGV-01 — DDNS static zone overwrite [FIXED]

**Implementation:** `src/dns/local.rs`, `src/dns/ddns.rs`

`LocalZoneSet` now carries `pub static_names: HashSet<Name>`, populated in
`from_config()` from all zone names and `local-data` record names.

`handle_update()` pre-checks all DELETE operations (class=ANY/255 or NONE/254) against
`static_names` before applying. On match, returns `ResponseCode::Refused` with a WARN
log entry. Static names are populated at startup and not modifiable at runtime.

---

### SEC-20 — TSIG key caching [FIXED]

**Implementation:** `src/dns/server.rs`

`RunboundHandler::new()` now accepts `tsig_keys_raw: Vec<(String, String, String)>` and
decodes all keys at startup into `tsig_keys: Vec<(String, TsigAlgorithm, Vec<u8>)>`.
The per-request path in `handle_update()` does no base64 decode or algorithm parsing.

---

### SEC-21 — Stale cache OOM [FIXED]

**Implementation:** `src/dns/server.rs`

Before inserting into the stale cache, the handler checks `sc.len() >= cache_max_entries`.
If at capacity, one entry is evicted (first key from `DashMap::iter()`). This is O(1)
amortized but not strict LRU. A proper LRU structure is deferred to v1.0 alongside
the planned stale cache refactor.

**Residual risk:** Eviction policy is not LRU — most-recently-used entries are not
preferentially retained. Acceptable for v0.9.4; proper LRU in v1.0.

---

### SEC-AGV-02 — Schedule bypass [FIXED]

**Implementation:** `src/api/mod.rs`, `src/store.rs`

`valid_hhmm()` validates HH:MM format (length 5, `:` at position 2, hours 0–23, minutes
0–59) before creating a `BlacklistEntry` with a schedule. Returns 400 on invalid input.

`is_active_now()` uses `self.start.get(..2)` / `self.start.get(3..5)` (returns `None`
on short strings, no panic) and range-checks parsed values.

---

### SEC-24 — DNSSEC stripping oscillation [FIXED]

**Implementation:** `src/upstreams.rs`

Added `consecutive_strip_count: u32` field (skipped by serde). The field increments on
each consecutive AD=0 probe result. `dnssec_stripping` is only set to `true` when
`consecutive_strip_count >= 2`. A single AD=1 result resets both the counter and the
flag. WARN log includes the current count.

---

### SEC-26 — SSE reconnect [FIXED]

**Implementation:** `examples/web-ui/index.html`

`scheduleSpkReconnect()` uses a `_spkBackoff` variable starting at 1000 ms, doubling
on each failure up to 30,000 ms maximum. Backoff resets to 1000 ms on successful
connection. Triggered on `fetch()` rejection, non-OK response, reader `done`, and
`reader.read()` rejection.

---

### SEC-AGV-03 — Stale after zone delete [DEFERRED v1.0]

Stale cache eviction on blacklist delete is not implemented. The stale cache will
continue to serve evicted records until `stale_max_age` expires or the entry is
naturally displaced. Impact is limited to the serve-stale path (only consulted on
upstream SERVFAIL).

---

### SEC-AGV-04 — API key in debug log [DOCS NOTE]

At `RUST_LOG=debug`, the `reqwest` client may log outbound request headers including
the `Authorization: Bearer <key>` injected by the WebUI proxy. Default log level is
`INFO` — not affected. Noted in configuration reference.
