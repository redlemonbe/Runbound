DNSSEC Validation

Validate DNSSEC signatures

When enabled, cryptographic signatures are validated on all responses. When disabled, the upstream AD bit is trusted.

Resolution Mode

How names are resolved

Forward sends queries to your upstream resolvers (DoT). Sovereign recurses from the IANA root servers with no third-party forwarder — DNSSEC-validated, anti-SSRF. In Sovereign mode the upstream resolvers are not used (the Upstreams tab is hidden); each node needs outbound port 53 to the internet. Admin only.

Local CA Certificate

Install this CA certificate once on each device to get a trusted HTTPS connection without browser warnings.

↓ Download CA certificate
  • macOSDouble-click the file → Keychain Access → right-click → Get Info → Trust → Always Trust
  • WindowsDouble-click → Install Certificate → Local Machine → Trusted Root Certification Authorities
  • Linuxsudo cp runbound-ca.pem /usr/local/share/ca-certificates/runbound.crt && sudo update-ca-certificates
  • FirefoxSettings → Privacy & Security → Certificates → View Certificates → Authorities → Import
  • iOSOpen the file → Settings → General → VPN & Device Management → Install → then Certificate Trust Settings → Enable
  • AndroidSettings → Security → Install from storage → CA Certificate
Encrypted DNS — DoT / DoH / DoQ
—

Serve DNS over TLS (853), HTTPS (443) and QUIC (853/udp) so clients such as Firefox can encrypt their queries to Runbound. Generate a self-signed certificate, or import your own (e.g. Let's Encrypt). Changes are applied live — no restart.

⚠ Configuration saved — restart Runbound to activate the new encrypted-DNS settings.
